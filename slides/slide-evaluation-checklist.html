Evaluation Checklist

Most Neglected

We'll build this step-by-step in Part 4, Habit 3.

Golden dataset of 200–500 test cases maintained in version control
Automated evaluation: LLM-as-judge + task-specific metrics + factual consistency
Hallucination rate measured and alerted
Regression tests on every prompt change, model update, or retrieval change
Human review queue for edge cases — someone reviews worst outputs daily
This is the most neglected category — say this with conviction. 70% of RAG systems have no systematic evaluation. The golden dataset of 200-500 test cases should be version controlled and maintained like code. Regression tests must run on every prompt change, model update, or retrieval pipeline change — not just on major releases. The human review queue means someone reviews the worst outputs daily, not weekly. We will build this step-by-step in Part 4 when we cover Habit 3 and the minimum viable evaluation stack.
