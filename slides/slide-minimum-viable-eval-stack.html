The Minimum Viable Evaluation Stack

Golden dataset: 100–300 representative queries with expected outputs. Versioned in git.
LLM-as-judge pipeline: Claude Opus 4.6 or GPT-5.4 evaluating accuracy, groundedness, completeness.
Factual consistency score: Does every claim appear in source documents?
Regression gate: Every PR that changes a prompt must pass evaluation. Score drops >3% = PR blocked.
The golden dataset is your ground truth — 100 to 300 representative queries with expected outputs, versioned in git, maintained as the system evolves. The LLM-as-judge pipeline uses a capable model like Claude Opus 4.6 to evaluate outputs on accuracy, groundedness, completeness, and format compliance. Factual consistency scoring checks whether every claim in the output appears in retrieved source documents — tools like RAGAS and TruLens help here. The regression gate is critical: every PR that changes a prompt, retrieval config, or model parameter must pass evaluation. If the score drops more than 3%, the PR is blocked. This is CI/CD for AI.
