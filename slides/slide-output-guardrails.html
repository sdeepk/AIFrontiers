Output Guardrails

Factual consistency scoring — does the response cite sources? Do claims match documents?
Harmful content filtering — block toxic, biased, or unsafe content before it reaches the user
PII detection in outputs — agents can accidentally surface data they shouldn't
Confidence expression — agents must be able to say "I don't have reliable information"
Factual consistency scoring should run before any output is displayed to the user — check whether the response cites sources and whether claims match the retrieved documents. PII detection in outputs is often overlooked: agents can accidentally surface sensitive data from retrieved documents that should have been filtered. The confidence expression requirement is crucial for trust — agents should always be able to say "I don't have reliable information on this" rather than confidently hallucinating. Make this a mandatory part of your output schema, not optional behaviour.
