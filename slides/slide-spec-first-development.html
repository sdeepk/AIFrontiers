Habit 4

Spec-First Development

The spec is not bureaucracy. The spec is the context that makes everything else better.

The spec is the context that makes AI assistance 10x better throughout the entire project. When you have a SPEC.md in the repo, every developer and every AI tool in the workflow shares the same context. You stop re-explaining the system to each other and to the AI. Onboarding a new developer becomes: read SPEC.md, open Cursor, start contributing. The spec also catches misunderstandings before any code is written, when they are cheapest to fix.
