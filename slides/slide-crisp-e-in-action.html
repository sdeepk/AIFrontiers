CRISP-E in Action

Bad Prompt

Write a login function

CRISP-E Prompt

You are a senior Node.js security engineer working on an Express API. Write a login handler that validates against users table using bcrypt, returns JWT signed with RS256, returns 401 without revealing which field is wrong, logs all attempts. I will verify with: valid user, invalid password, non-existent email, SQL injection attempt.
Walk through the right-hand example: notice it specifies Context (Express API with PostgreSQL), Role (senior Node.js security engineer), Intent (login handler), Specifics (bcrypt, RS256 JWT, 401 without revealing which field, audit logging, no new dependencies), Prior art (references existing db client), and Evaluation (test with valid user, invalid password, non-existent email, missing fields, SQL injection). The bad prompt produces generic code. The CRISP-E prompt produces production-ready, security-conscious code on the first attempt. This is a real example — try it yourself after this session.
