The Agentic RAG Architecture

1 Router Retrieve or not? Which method?
→
2 Retriever Semantic, keyword, graph, or hybrid
→
3 Grader Do docs answer the query?
→
4 Generator Answer with citations
→
5 Hallucination Checker Verify claims against sources
↩ If grade = insufficient → rewrite query and retrieve again
Walk through each component: the Router decides whether to retrieve at all and which method to use. The Retriever executes semantic, keyword, or graph search — or all three in parallel. The Grader evaluates whether retrieved documents actually answer the query — if not, it rewrites the query and retrieves again (the feedback loop at the bottom). The Generator produces the answer with citations per claim. The Hallucination Checker verifies every claim against source documents before the user sees it. This feedback loop is what distinguishes agentic RAG from basic RAG — it self-corrects before responding.
