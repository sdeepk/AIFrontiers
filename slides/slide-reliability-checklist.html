Reliability Checklist

Retry logic, circuit breakers, and fallback model routing configured
Max iteration limits enforced for agents (prevent runaway loops)
Latency p95 within user tolerance (interactive: <3s)
Agent state is recoverable — can resume or explain on failure
Tool call permissions explicitly whitelisted
Two items to stress verbally: tool call whitelisting means agents should never have open-ended API access to production systems — this is a common mistake. Agent state recoverability means if an agent fails mid-task, it should be able to resume or at minimum explain what it completed before failing. For latency, the rule of thumb is interactive under 3 seconds at p95 and batch workloads on acceptable async timelines. Max iteration limits are non-negotiable — we will see in Section 2.5 what happens when a loop runs away.
