Standardise the Toolkit

Action 1
One agentic coding IDE (Cursor or VS Code + Claude Code)
One conversational AI (Claude)
One evaluation framework (RAGAS or TruLens)
The diversity tax eliminates all compounding benefits
The diversity tax is real: when every developer uses a different AI tool with no shared practices, no shared prompts, and no shared context, you lose all the team-level compounding benefits. Pick one tool per category and standardise. Then build shared prompt libraries, shared context files, and shared evaluation frameworks on top of the standard. The specific recommendations here — Cursor for coding, Claude for conversation, RAGAS or TruLens for evaluation — are current best-of-breed for 2026. The choice matters less than the standardisation.
