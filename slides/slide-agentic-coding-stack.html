The Agentic Coding Stack

Deep Dive
▸
Rules: Project-level instructions (CLAUDE.md, .cursorrules, AGENTS.md) that shape agent behavior per-repo — coding conventions, architectural constraints, what to avoid
▸
Skills: Reusable prompt templates (slash commands) that encode team workflows — /review, /test, /deploy — parameterized and shareable across the team
▸
Hooks: Shell commands triggered on agent events (pre-tool-use, post-tool-use) — enforce guardrails, run linters, log actions, gate destructive operations
▸
Workflows: Multi-step agentic loops — plan, implement, test, fix — with human checkpoints, spec-first development, and CI integration
▸
Key insight: MCP connects agents to tools. Rules, skills, hooks, and workflows make agents reliable and team-aware.
This is the layered system that makes coding agents team-aware and reliable. Rules files like CLAUDE.md and .cursorrules are committed to the repo so every developer and agent follows the same conventions. Skills are reusable prompt templates shared across the team. Hooks enforce guardrails automatically — run linters, gate destructive operations. Workflows tie it all together with human checkpoints and CI integration. This stack is what separates toy demos from production coding agents.
