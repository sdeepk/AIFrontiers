Contextual Retrieval

Technique: Before indexing, generate a context summary for each chunk
Example: 'This chunk comes from Document X, section Y, about topic Z'
Chunks indexed with context are retrieved with dramatically higher precision
This is an Anthropic technique that dramatically improves retrieval precision. Before indexing, you use a model to generate a context summary for each chunk explaining where it comes from and what it discusses in the context of the larger document. The model then has enough context during retrieval to know when a chunk is actually relevant versus just superficially similar. This is cheap to implement — it is a one-time indexing cost — and the precision improvement is substantial. If you are building any RAG pipeline, add contextual retrieval to your indexing step.
